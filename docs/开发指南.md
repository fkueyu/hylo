# Hylo 开发与贡献指南

感谢你参与 Hylo 的开发！本指南将帮助你快速设置开发环境并了解项目的开发流程。

## 1. 环境准备

Hylo 构建在现代的前端和系统级技术栈之上，你需要确保你的开发机器上已经安装了以下底层工具：

1. **Node.js**: v18+ (推荐使用最新的 LTS 版本)
2. **Rust**: 最新稳定的 Rust 版本 (`rustc` 和 `cargo`)
3. **平台构建工具**:
   - macOS: Xcode 或 Xcode Command Line Tools
   - Windows: Visual Studio C++ Build Tools
   - Linux: `webkit2gtk` 等构建依赖

## 2. 启动开发服务器

克隆仓库并安装依赖后，只需执行以下命令即可全自动拉起包含前端和后端的热更新开发环境：

```bash
# 1. 安装前端依赖
npm install

# 2. 启动开发环境
# 该命令会自动运行 `vite dev` 并在编译完成后呼出 Tauri Native 窗口
npm run tauri dev
```

> **注意**：由于 Tauri 构建时需要调用 `cargo`，如果遇到类似 `No such file or directory (os error 2)` 的环境变量错误，请确保 `~/.cargo/bin` 已经加入到你的终端 `PATH` 中。

## 3. 项目目录结构

```text
Hylo/
├── src/                    # React 前端代码
│   ├── components/         # 核心 UI 组件 (MonacoPanel, PreviewPanel, SplitView, ContextMenu)
│   ├── core/               # 核心业务逻辑引擎 (ast-parser, node-map, sync-manager, event-bus)
│   ├── hooks/              # 自定义 React Hooks (useEditor, useFileOpen, useFileSave, useContextMenu)
│   ├── i18n/               # 中英双语映射字典 (index.ts)
│   ├── types/              # 全局 TypeScript 类型定义 (index.ts)
│   └── App.tsx             # 应用主入口，负责处理全局右键、事件代理、布局状态
├── src-tauri/              # Rust 后端与系统集成代码
│   ├── src/
│   │   ├── main.rs         # 程序入口（仅调用 lib.rs）
│   │   └── lib.rs          # 主要业务逻辑：原生菜单构建、IPC 事件转发
│   ├── .cargo/
│   │   └── config.toml     # Rust 链接器配置（嵌入 Info.plist到二进制）
│   ├── capabilities/       # Tauri v2 权限管控名单 (读写控制)
│   ├── Info.plist          # macOS 应用属性声明（语言、区域等）
│   └── tauri.conf.json     # Tauri 核心配置信息
├── docs/                   # 项目文档目录
└── package.json            # 前端依赖配置
```

## 4. 关键规范与修改须知

- **UI 增减原则**：我们推崇极简主义，绝不在主界面增加任何挤占“垂直空间”的冗余按钮。所有的额外工具选项，请优先集成至左右面板的 `headerRight` 区域，或是沉淀入**全局右键菜单**之中。
- **文件保存逻辑**：修改 `src-tauri/capabilities/default.json` 是赋予 Tauri 应用全新底层能力（如网络请求、新目录的读写权限等）的唯一途径，开发相关功能时切勿忘记注册 Capability 权限。
- **多语言强制性**：任何面向用户的字符串必须录入 `src/i18n/index.ts` 字典中，**中英文两个 locale 必须同步添加**，严禁在 UI 组件中硬编码中文或英文字符串。
- **防中断 Monaco 劫持**：如需扩展 Monaco Editor 的交互或右键，请永远保持在 `MonacoPanel.tsx` 中向 Editor 传入 `contextmenu: false`，然后使用外层的 React `onContextMenu` 来实现事件收口。
- **新建文件行为**：新建操作将清空编辑器为空白内容，而非加载预置模板。若需恢复初始模板行为，请修改 `App.tsx` 中 `handleNewFile` 函数里的 `editor.loadContent("")` 为具体内容。
- **防丢失确认逻辑**：新建、打开文件前的数据丢失确认机制集中在 `App.tsx` 的 `handleNewFile` 和 `handleOpenFile` 中，依赖 `editor.isDirty` 状态判断，使用 `@tauri-apps/plugin-dialog` 的 `ask()` API 弹出系统对话框。
