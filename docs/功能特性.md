# Hylo 功能特性与用户手册

Hylo 致力于将 HTML 从枯燥的代码字符串还原为可见、可触、可交互的 DOM 树节点，并打造如同编写 Markdown (Typora) 一样流畅的编辑体验。

## 核心特性列表

### 🍎 纯粹的 macOS Native 体验
- **零冗余的无边框布局**：移除了传统应用的顶部标题栏，极简界面，彻底释放你宝贵的垂直屏幕空间，全屏沉浸式打码。
- **原生应用级响应**：系统原生的打开/保存对话框机制，基于 Rust 编写的底层，跳过浏览器沙盒直接以极高效率执行原生系统文件 I/O，并原生支持 `Cmd+S`, `Cmd+O` 等快捷键。
- **系统级菜单桥接**：所有的快捷操作、语言切换均完美同步挂载于 macOS 的顶部系统原生级菜单栏。

### ⚡ 双向时光隧道联动 (Live Sync)
无论是编写长篇的详情页还是复杂的嵌套 UI，你再也不会“迷路”：
- **预览溯源**：在右侧的视觉预览面板中，遇到任何想要修改的元素，只需“点击”它，左侧的 Monaco 编辑器就会如同施放了魔法一般，瞬间定位并滚动至该元素对应的确切代码行。
- **源码投影**：当你的输入光标在左侧层层嵌套的 HTML 标签中移动时，右侧的可视化区域也会实时高亮展示你当前正在编写的 DOM 结构究竟在哪里。

### 🛡️ 所见即所隔离的沙盒预览
- 预览区完全运行于 **Shadow DOM** 内，这意味着你可以放心地编写包含 `<style>` 全局注入、`body` 覆写甚至是 `CSS Reset` 代码的完整 HTML 片段，它们绝对不会污染或破坏 Hylo 应用本身的 UI 界面。

### 🌐 细节至上的智能生态
- **双语支持**：内置完善的 i18n 支持，一键即可全局在中文 / 英文界面间顺滑切换。
- **暗黑/明亮双轨主题**：适配夜间和白天的编码习惯，拥有精调的 `vs-dark` 和亮色代码高亮风格。
- **防丢失状态护航**：智能监控文件脏状态 (Dirty State Tracking)，文件修改后标题栏实时显示高亮 `*` 号；执行新建或打开文件等破坏性操作前，会弹出系统级二次确认对话框，双重防止数据丢失。
- **三栏布局自由切换**：顶部右侧提供布局切换器，支持「双栏 / 仅源码 / 仅预览」三种模式一键切换，适应不同工作场景。
- **现代右键生态**：抛弃了经常被浏览器拦截失效的古老方案，基于底层 `navigator.clipboard` 重写的全功能中文全局右键菜单，真正实现了顺滑的跨平台复制、剪切、粘贴体验。

## 快速上手

1. 启动 Hylo 后，你将立刻看到一份关于 Hylo 的自述介绍页面源码，左侧为可编辑的 HTML 源码，右侧为实时渲染预览。
2. 右键任意位置，可在右键菜单中选择**新建**（清空编辑器开始新文档）或**打开文件**（载入本地 `.html` 文件）。
3. 对光标处的代码感到混乱？右键呼出**格式化代码**（`Format Document`）进行一键梳理。
4. 按下 `Cmd + S` 即可保存；如果是新文件，系统会弹出另存为对话框，并自动预填 `无标题.html`。
5. 通过顶部右侧的**布局切换器**，可在「双栏」「仅源码」「仅预览」三种视图模式间快速切换。

> **注意**：在执行新建或打开文件时，若当前文件有未保存的修改，系统将弹出确认对话框，防止意外丢失工作成果。
