## 一、产品定位

**Hylo** 是一款面向 macOS 的 **AI 原生 HTML 可视化编辑器**。

它的核心主张是：

> *"HTML 的 Typora"* — 将 HTML 文档从枯燥的代码字符串，还原为可见、可触、可交互的结构化内容，像写 Markdown 一样流畅地编辑 HTML。

### 背景与动机

随着大语言模型（LLM）能力的提升，AI 生成 HTML 已变得极为普遍。然而现有工具存在严重的体验割裂：

- 传统 IDE（VS Code、WebStorm）专为"代码工程师"设计，无法提供 HTML 内容的"实时渲染预览"与"点击溯源"能力。
- 浏览器开发者工具只能检查，无法直接编辑源文件。
- 在线 HTML 编辑器（如 CodePen）需要联网，且缺乏本地文件系统集成。

**Hylo 的切入点**：在本地运行，原生速度，双栏实时联动，让用户"看着渲染结果写代码"。

---

## 二、目标用户

| 用户群体 | 核心诉求 |
|---------|---------|
| **AI 内容创作者** | 快速审阅和精调 LLM 生成的 HTML 文档或邮件模板 |
| **前端开发者** | 在编写 HTML 片段时获得即时视觉反馈，减少 F5 次数 |
| **内容设计师** | 无需深度理解 HTML 标记，通过"所见即所得"方式调整内容结构 |
| **独立技术作者** | 快速编写和预览用于发布的单页 HTML 文档或报告 |

**目标平台**：macOS 12+（Apple Silicon 与 Intel 原生支持）

## 三、产品目标

### 阶段 1（当前 v0.1）：建立核心体验闭环

- [x] 实现 HTML 源码与渲染预览的**双向实时联动**
- [x] 提供流畅的**本地文件读写**能力（打开 / 保存 / 新建）
- [x] 打造符合 macOS HIG 规范的**原生视觉体系**
- [x] 支持**暗色 / 亮色主题**切换
- [x] 支持**中英文**界面语言切换
- [x] 实现**未保存状态保护**（脏状态追踪 + 二次确认弹窗）

### 阶段 2（待规划）：增强 AI 协作能力

- [ ] 接入 LLM API，支持"选中代码段 → 描述意图 → AI 修改"的行内 AI 编辑
- [ ] 支持从剪贴板直接粘贴完整 HTML 文档并即时渲染
- [ ] 支持多标签页（Tab）同时打开多份文档
- [ ] 提供大纲视图（DOM 树侧边栏），展示 HTML 文档的层级结构

### 阶段 3（远期愿景）：扩展平台能力

- [ ] 开放 Plugin API，支持社区扩展
- [ ] 支持 CSS / JavaScript 文件的联动预览
- [ ] 提供 Windows / Linux 版本

## 五、非功能需求（Non-functional Requirements）

| 类别 | 要求 |
|------|------|
| **性能** | 实时解析延迟 ≤ 300ms（防抖后），AST 更新不阻塞主线程 |
| **稳定性** | 新建 / 保存 / 打开操作中，任何路径下均不可发生数据丢失 |
| **安全** | 所有文件 I/O 通过 Tauri Capabilities 白名单管控，不暴露任意路径访问权限 |
| **响应性** | 窗口最小尺寸 800×600，支持全屏模式 |
| **可扩展性** | i18n 字典统一维护，新增语言仅需添加翻译表，无需修改组件代码 |
| **兼容性** | 支持 macOS 12+ (Monterey)，同时面向 aarch64 (Apple Silicon) 和 x86_64 (Intel) |

## 六、技术约束与边界

1. **仅限本地运行**：不包含任何网络请求、账号体系或云同步功能（当前版本）。
2. **不执行脚本**：`<script>` 标签内的 JavaScript 代码在预览区不执行，仅渲染结构。
3. **单文件编辑**：当前版本一次只支持一份打开的文档，不支持多标签或工作区概念。
4. **不处理资源引用**：`<img src="...">` 等外部资源引用若指向本地路径，受 Tauri CSP 限制可能无法加载。

---

## 七、设计原则

1. **极简主义**：主界面不增加任何占用垂直空间的额外控件，保持双栏内容区最大化。
2. **原生优先**：遵循 Apple Human Interface Guidelines，优先使用系统原生字体、颜色与交互行为。
3. **操作直觉性**：用户操作路径应与 macOS 常见应用保持一致（Cmd+S 保存、拖拽调整等），减少学习成本。
4. **安全第一**：任何会导致数据丢失的操作（新建、打开、关闭）都必须设置二次确认保护。
