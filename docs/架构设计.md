# Hylo 系统架构设计

Hylo 是一款定义为“HTML 的 Typora”的 AI 原生可视化编辑器。本文档详细阐述了其底层的技术选型与核心模块设计。

## 核心技术栈

- **前端框架**: React 19 + TypeScript + Vite
- **底层驱动**: Tauri v2 + Rust
- **代码编辑器**: Monaco Editor (`@monaco-editor/react`)
- **AST 解析引擎**: `parse5` (纯粹的 HTML5 标准解析引擎)
- **多语言支持**: 内部极简的 i18n 机制 (中/英)

## 系统核心模块

### 1. 双向联动引擎 (SyncManager & NodeMapManager)
Hylo 的核心灵魂在于“源码”与“预览”的无缝联动：
- **`parseHTML` (ast-parser.ts)**: 利用 `parse5` 将用户输入的 HTML 实时解析为抽象语法树 (AST)，并生成节点标识（`__hylo_id__`）。
- **`NodeMapManager`**: 维护一张巨大的映射表，记录每一个 `__hylo_id__` 在 Monaco 编辑器中的精确起止行号 (SourceLocation)。
- **双向同步交互**:
  - **点击预览树节点** -> 查询 NodeMap -> 驱动 Monaco 的 `editor.revealPositionInCenter` 并施加 `deltaDecorations` 实现代码区域高亮。
  - **代码区光标移动** -> 查询 NodeMap -> 定位最内层 AST 节点 -> 通过事件总线触发预览区高亮。

### 2. 沙盒隔离渲染机制 (PreviewPanel)
为了防止用户输入的 HTML 样式（如 `body { background: black }` 或 CSS 重置）污染整个应用的外围 React UI，右侧预览区使用了 **Shadow DOM 隔离渲染**：
- React 通过 `createPortal` 直接将由 AST 逆向渲染生成的 UI 投射进入 `attachShadow({ mode: "open" })` 的内部根节点。
- 所有全局样式、重置样式都被限制在 Shadow Boundary 内，从而实现了真正的所见即所得。

### 3. Tauri v2 底层融合桥接
Hylo 不是一个简单的套壳 Web App，而是深度整合了系统级底层能力的 Native 应用：
- **原生应用交互**: 通过 `@tauri-apps/plugin-fs` 跨过浏览器沙盒直接对本地硬盘进行原生的 I/O 读写。
- **全局原生菜单联动**: 在 Rust 侧 (`src-tauri/src/lib.rs`) 构建基于操作系统的系统原生菜单栏，并在点击时通过跨进程事件总线 (IPC `app_handle.emit`) 向 React 发送命令（如 `save-file`、`toggle-theme`）。
- **macOS 本地化深度适配**: 通过嵌入 `Info.plist`（输入 `-Wl,-sectcreate` 链接器属性）将应用的语言声明固化到二进制文件内，确保系统原生对话框（保存/打开面板）在正式打包后能正确显示中文。

### 4. 彻底重构的右键系统 (ContextMenu)
为解决 Monaco Editor 与 Tauri/WebKit 环境下复杂的右键事件劫持冲突，Hylo 实现了一套“降维”接管的右键生态：
- 通过强制在 Monaco 内核传递 `contextmenu: false` 关闭内置劫持。
- 在全局的 `.app`、`.monaco-panel__editor` 容器上直接绑定 `onContextMenu` 拦截系统级右键事件（辅以防冒泡逻辑）。
- 所有复制、剪切、粘贴绕过了陈旧的 `document.execCommand` 限制，直接对接了现代异步的 `navigator.clipboard` 系统级剪贴板 API。
